# EntityData 重构方案自审

## 覆盖检查
- [x] 枚举分层：`EntityType` + 分类枚举 + `Max`。
- [x] 数据分层：中枢索引 -> 分类桶 -> 类型包体。
- [x] 编辑器自动地址维护：创建/保存时自动写Addressables。
- [x] 删除前引用扫描阻断。
- [x] 单项+全量校验，含重复address检测。
- [x] 单体编辑器按类型拆分。
- [x] 蓝图交互增强（新建后自动打开、编辑按钮、库折叠、改名）。
- [x] 运行时反向查询与变更通知保留。

## 风险与修正
1. **风险：旧调用仍引用全局 `EntityId`**
   - 处理：保留生成文件路径不变，内部改为三个枚举；服务接口已切换为分类方法。
2. **风险：Addressables Editor API 在不同环境可用性不一致**
   - 处理：继续反射调用，避免编译时硬依赖。
3. **风险：删除引用扫描遗漏字段**
   - 处理：当前覆盖 icon/preview/sfx 三类地址字段；后续新增地址字段时必须在扫描器中同步注册。
4. **风险：蓝图重命名引发路径冲突**
   - 处理：当前采用保存覆盖策略，若目标同名存在依赖 `SaveFile` 反馈，避免静默覆盖失败。

## 结论
- 方案覆盖本轮全部需求点，具备可落地代码路径。
- 下一阶段重点是完善迁移脚本和更多类型字段级校验规则。
