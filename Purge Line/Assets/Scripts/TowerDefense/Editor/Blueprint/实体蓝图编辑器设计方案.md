# 实体蓝图编辑器设计方案

## 1. 目标与边界
- 目标：提供一个基于 Unity UI Toolkit 的可视化实体蓝图编辑器，用于创建/编辑/保存/验证 ECS Entity 模板。
- 版本：Unity 2022.3 LTS，Entities 1.4.5+（当前工程可先兼容 1.3.x API 子集，代码按 1.4+ 约束设计）。
- 非目标：不实现节点行为逻辑，不依赖第三方编辑器框架。

## 2. 整体架构设计图

```text
+--------------------------------------------------------------------------------+
| EntityBlueprintEditorWindow (UI Toolkit)                                       |
|--------------------------------------------------------------------------------|
| Toolbar: New/Open/Save/Create/RunTest/Validate + recent files                 |
| Left Pane: Search + Favorites + Category Tree                                 |
| Main Pane: Reorderable Component Cards (Foldout + Field Editors)              |
| Bottom: Status Bar + Internal Console                                          |
+---------------------------+---------------------------+------------------------+
                            |                           |
                            v                           v
         +----------------------------------+   +------------------------------+
         | EntityBlueprintEditorState       |   | ComponentCatalog/Drawer      |
         | - current document               |   | - type scan & classification |
         | - dirty state/recent/autosave    |   | - typed field rendering      |
         +-------------------+--------------+   +---------------+--------------+
                             |                                  |
                             v                                  v
                 +--------------------------+       +----------------------------+
                 | EntityBlueprintDocument  |       | ReflectionTypeBridge       |
                 | components/fields/order  |       | value<->string conversion  |
                 +------------+-------------+       +-------------+--------------+
                              |                                   |
                              v                                   v
                 +---------------------------+      +-----------------------------+
                 | BinarySerializer          |      | RuntimeFactory              |
                 | save/load/versioning      |      | instantiate to EntityManager|
                 +---------------------------+      +-----------------------------+
```

## 3. 模块划分与职责

### 3.1 Runtime（`Assets/Scripts/TowerDefense/Data/Blueprint`）
1. `EntityBlueprintDocument`
   - 蓝图数据根对象（组件列表、字段数据、元信息、格式版本）。
2. `EntityBlueprintBinarySerializer`
   - 自定义二进制序列化/反序列化（高性能、低 GC）。
3. `EntityBlueprintRuntimeFactory`
   - 将 `EntityBlueprintDocument` 实例化到 `EntityManager`。
4. `EntityBlueprintAuthoring`
   - SubScene/Authoring 桥接；可将蓝图文件路径烘焙为实体组件。

### 3.2 Editor（`Assets/Scripts/TowerDefense/Editor/Blueprint`）
1. `EntityBlueprintEditorWindow`
   - UI Toolkit 主窗口、交互入口、命令调度。
2. `EntityBlueprintEditorState`
   - 编辑态数据、脏标记、自动保存、最近文件、布局锁定状态。
3. `EntityBlueprintComponentCatalog`
   - 扫描 ECS 组件类型，分类树、搜索、收藏。
4. `EntityBlueprintFieldDrawer`
   - 按字段类型动态生成输入控件；支持 int/float/bool/Vector/FixedString64Bytes/Entity。
5. `EntityBlueprintValidationWindow`
   - 展示序列化前后 diff 结果，高亮差异。
6. `EntityBlueprintEditor.uss`
   - UI 样式定义（卡片、状态栏、控制台、分割布局）。

## 5. 序列化格式定义

- 文件后缀：`.entitybp`
- 二进制布局：
  1. 4 bytes magic = `EBP1`
  2. `int formatVersion`
  3. `string blueprintName`
  4. `int componentCount`
  5. 每个组件：
     - `string componentTypeName`
     - `string category`
     - `int fieldCount`
     - 每个字段：`string fieldPath` + `string fieldTypeName` + `string serializedValue`

兼容策略：
- `FormatVersion` 前向递增；旧版本通过 `switch(version)` 做迁移。
- 未识别组件/字段跳过并写入 warning，不阻塞整体加载。

## 6. UI 布局设计

1. 顶部工具栏
   - `新建` `打开` `保存` `创建实例` `▶ 运行测试` `验证数据` `布局锁定`
2. 最近文件条
   - 显示最近 5 个文件按钮，点击快速打开。
3. 左侧组件库
   - 搜索框、收藏夹、分类树（可折叠）。
4. 主面板组件卡片
   - 每个组件 `Foldout` + 删除按钮 + 字段编辑区。
   - 支持拖拽排序（锁定布局时禁用）。
5. 底部区域
   - 状态栏：当前路径、保存状态、组件数。
   - 内置 Console：操作日志滚动列表。

## 7. 数据流（编辑 -> 保存 -> 加载 -> 实例化）

1. 编辑
   - 用户从组件库添加组件 -> 创建 `ComponentRecord` -> 反射提取字段默认值。
2. 保存
   - Window 调用 `EntityBlueprintBinarySerializer.Save` 写入 `.entitybp`。
3. 加载
   - 打开文件 -> `Load` -> 替换当前 Document -> 刷新 UI。
4. 实例化
   - `Create` 按钮调用 `EntityBlueprintRuntimeFactory.Create`：
     - 在当前 World 创建实体
     - 添加组件并回填字段
     - 位置默认 `(0,0,0)`，测试流程自动聚焦。
5. 验证
   - `Validate` 执行 `Save->Load`（内存 round-trip）
   - 对比 source 与 loaded，差异展示在 `EntityBlueprintValidationWindow`。

## 8. 性能优化方案

1. 序列化
   - 使用 `BinaryWriter/BinaryReader`，避免 JSON 解析开销；字符串池化可后续扩展。
2. UI
   - 组件库使用虚拟化 `ListView`，搜索时仅刷新数据源。
3. 反射缓存
   - `Type -> FieldInfo[]` 缓存，减少重复反射。
4. 自动保存
   - 仅在 dirty 且间隔到达时保存，避免频繁 IO。
5. Diff 验证
   - 使用结构化字段比对，按组件/字段维度输出最小差异。

### 9.2 架构风险与修正
1. 风险：SubScene 直接写入实体复杂且版本相关。
   - 修正：首版通过 Authoring 桥接写入 SubScene（工业项目常见做法，稳定）。
2. 风险：反射字段类型扩展时侵入核心窗口。
   - 修正：字段渲染下沉到 `EntityBlueprintFieldDrawer` 注册表，新增类型无需改窗口主流程。
3. 风险：组件类型重命名导致加载失败。
   - 修正：保存 assembly-qualified type + 兼容 fallback；在控制台提示并允许跳过。
